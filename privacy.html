개인정보 처리 및 AI 이용 안내

KIBU가 학생 자료를 어떻게 다루고, 무엇이 어디로 전송되는지 투명하게 안내합니다.

최종 업데이트: 2026.07.04

한눈에 보기
KIBU는 선생님의 PC에서 구동되는 설치형 보조도구입니다. 학생 자료와 생성된 초안은 KIBU 서버가 아닌 선생님 PC(로컬)에만 저장됩니다. 다만 초안 생성을 위해 자료의 일부가 선생님이 직접 등록한 OpenAI / Google(Gemini) API를 통해 해당 사업자(국외)로 전송됩니다. 개인정보 보호의 최종 책임 주체(개인정보처리자)는 이를 사용하는 학교·교사이며, KIBU는 그 처리를 돕는 도구입니다.

로컬 우선 저장

학생 자료·초안은 교사 PC의 로컬 데이터베이스에만 저장되며, KIBU 자체 서버로 수집·전송하지 않습니다.

AI 처리는 국외 전송

초안 생성 시 자료 텍스트가 OpenAI(미국)·Google(미국) 등 AI 사업자 서버로 전송·처리됩니다.

내 API 키 · 암호화 보관

교사 본인의 API 키로만 구동되며, 키는 Windows DPAPI로 암호화되어 로컬에 저장됩니다.

이미지 전송은 선택(기본 꺼짐)

스캔본 원본 이미지는 기본적으로 전송되지 않으며, 설정에서 직접 켠 경우에만 정밀 판독용으로 전송됩니다.

1. 데이터가 어디로 가나요?

KIBU에서 다루는 데이터는 크게 로컬에만 머무는 것과 AI 처리를 위해 외부로 전송되는 것으로 나뉩니다.

데이터처리 위치설명
학생 자료 원본·명렬표·생성된 초안 로컬 교사 PC의 로컬 DB(SQLite)에 저장. KIBU 서버로 전송되지 않습니다.
API 키(OpenAI / Gemini) 로컬(암호화) Windows DPAPI로 암호화하여 로컬에만 저장. 해당 키는 각 AI 사업자 인증에만 사용됩니다.
초안 생성용 텍스트(활동 내용, 평가 맥락 등) 국외 전송 초안 생성을 위해 OpenAI / Google API로 전송·처리됩니다.
스캔본 원본 이미지(손글씨 등) 선택(기본 꺼짐) 기본은 로컬 OCR 텍스트만 사용. [설정 → 외부 이미지 분석]을 켠 경우에만 이미지가 전송됩니다.

2. 국외 이전 안내

KIBU는 자체 AI 모델을 두지 않고, 선생님이 등록한 API 키를 통해 아래 사업자의 생성형 AI 서비스를 이용합니다. 초안 생성 과정에서 자료의 일부가 이 사업자의 서버(대부분 국외)로 전송됩니다.

  • OpenAI, L.L.C. — 이전 국가: 미국 등 / 이전 항목: 초안 생성에 사용되는 텍스트(및 이미지 옵션 활성화 시 이미지) / 이용·보유: OpenAI의 API 데이터 처리 정책에 따름
  • Google LLC (Gemini API) — 이전 국가: 미국 등 / 이전 항목: 동일 / 이용·보유: Google의 데이터 처리 정책에 따름
Gemini 무료 등급 유의
Google Gemini의 무료(Free) API 등급은 입력 데이터가 서비스 품질 개선(모델 학습 등)에 활용될 수 있습니다. 학생 개인정보가 포함될 수 있는 업무에는 유료(paid) 등급 사용 또는 데이터 미학습 정책이 적용되는 키 사용을 권장합니다. OpenAI API는 기본적으로 입력을 모델 학습에 사용하지 않습니다.

3. 개인정보 최소화와 마스킹

KIBU는 외부로 전송되는 개인정보를 줄이기 위해 다음을 적용합니다.

  • 이름·전화번호·이메일 등 직접 식별정보는 전송 전 마스킹 규칙으로 최소화합니다.
  • 한 학생의 초안을 만들 때 다른 학생의 이름이 섞여 들어가지 않도록 분리·치환합니다.
  • 스캔본 원본 이미지는 기본적으로 전송하지 않고, 로컬 OCR로 추출한 텍스트만 사용합니다.
다만 활동 내용 본문 안에 적힌 개인정보(예: 자기소개서 속 주소·가족관계 등)까지 자동으로 완벽히 제거되지는 않습니다. 업로드 자료에는 초안 작성에 꼭 필요한 정보만 남겨 주시길 권장합니다.

4. 저장 및 보안

  • 로컬 저장: 모든 학생 자료와 초안은 교사 PC 로컬에 저장됩니다. KIBU는 별도의 수집 서버를 운영하지 않습니다.
  • 키 암호화: API 키는 Windows DPAPI로 암호화되어 저장되며, 화면·로그에 평문으로 노출되지 않습니다.
  • PC 보안 책임: 자료가 로컬에 저장되므로, 공용/공유 PC 사용 시 계정 잠금·디스크 암호화 등 PC 자체 보안이 중요합니다.

5. 학교·교사(개인정보처리자)의 역할

학생 개인정보를 수집·이용하는 주체는 학교·교사이며, KIBU는 그 업무를 돕는 도구입니다. 안전한 활용을 위해 아래를 권장합니다.

  • 학교의 개인정보 보호 방침 및 생성형 AI 활용 지침을 확인하고 준수해 주세요.
  • 필요 시 학생·보호자에게 AI 보조도구 이용 사실과 국외 이전을 안내·동의받는 절차를 검토해 주세요.
  • 민감정보(건강·가족사항 등)는 가능한 한 업로드하지 않거나 사전에 제거해 주세요.
  • 생성된 초안은 반드시 교사가 검토·수정한 뒤 사용해 주세요. AI 결과는 초안 제안일 뿐 최종 기록이 아닙니다.

6. 프로그램 내 안내

KIBU는 최초 실행 시 위 내용을 요약한 개인정보·AI 이용 고지를 표시하며, 확인 후에만 사용을 시작할 수 있습니다. 또한 자료 업로드 화면과 설정 화면에서 관련 안내와 옵션(외부 이미지 분석 등)을 확인할 수 있습니다.

7. 국가 가이드라인·교원 역량체계와 KIBU

KIBU는 교사가 AI를 안전하고 책임 있게 활용하도록, 국가가 제시한 지침과 교원 역량 방향에 맞춰 설계되었습니다. 특정 기술을 자랑하기 위해서가 아니라, 아래와 같은 공통된 원칙을 도구 안에 담았습니다.

국가 개인정보·AI 보안 가이드라인이 요구하는 것

「생성형 AI 개발·활용을 위한 개인정보 처리 안내서」(개인정보보호위원회, 2025.8)와 「국가·공공기관 AI보안 가이드북」(국가정보원, 2025.12)은 외부 AI를 쓸 때 다음을 강조합니다. KIBU는 각 항목을 이렇게 반영합니다.

가이드라인이 권고하는 것KIBU의 반영
개인정보 최소화 · 입력/출력 단계 필터링 이름·연락처 등 식별정보를 AI 전송 전 마스킹, 초안 본문에도 실명이 남지 않도록 정리
사람이 최종 확인하는 절차(자동화된 결정 방지) AI는 초안만 제안하고, 검토 카드에서 교사가 확인·확정 — 완전 자동 결정이 아님
사용 이력 기록 · 투명한 고지 사용 모델·근거·검토 결과를 로컬 이력에 저장, 최초 실행 시 개인정보·AI 이용 고지
민감정보 입력 유의 · 외부 AI 설정 관리 업로드 자료 최소화 안내, Gemini 무료 등급·모델 학습 관련 유의사항 안내(위 2항 참고)
쉽게 말해, KIBU는 자료를 그대로 챗봇에 붙여넣는 방식 대신 가려서 보내고, 근거로만 쓰고, 여러 번 검토하고, 선생님이 확정하는 흐름을 도구 안에 담았습니다.

교원 AI·디지털 교육 역량체계와의 연결

교육부는 「교원 AI·디지털 교육 역량체계」(2026)를 통해, 교사에게 필요한 역량이 단순한 도구 사용법에서 AI를 활용한 수업·평가 설계와 AI 윤리·책임으로 옮겨가고 있다고 밝혔습니다. (국제 기준인 UNESCO 교사 AI 역량 프레임워크도 '인간 중심·AI 윤리'를 핵심에 둡니다.)

  • AI 윤리·기초 소양: 개인정보를 가리고 근거에 기반해 쓰는 KIBU의 흐름 자체가 윤리 원칙의 실천 사례입니다.
  • 과정 중심 평가·근거 활용: 수행·탐구·피드백 자료를 근거로 넣으면, 근거에 충실한 기록으로 이어집니다.
  • AI를 검증·책임지는 교사: KIBU에서 익히는 것은 '생기부를 빨리 쓰는 법'이 아니라 AI의 결과를 검토하고 책임지는 태도입니다.
