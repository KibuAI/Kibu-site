Core Features

생기부 업무 전체를 자동화합니다

단순 요약 AI가 아닙니다. 자료 분류부터 초안 생성, 규칙 검토까지 한 흐름으로 이어집니다.

One-Pass 맥락 분석

자료를 단순 OCR로 요약하지 않습니다. 자료 묶음을 한 번에 읽고 활동의 주제·진행 단계·학생의 고유한 기여를 종합적으로 구조화합니다.

구조화된 결과

자료 자동 배분

명렬표와 섞인 스캔본 하나만 올리면 학번 기준으로 학생별 보관함에 정확히 분류됩니다.

학생 자료 혼입 방지

여러 명을 동시에 처리해도 학생별로 완전히 분리된 공간에서 처리되어 자료가 섞이지 않습니다.

일괄 생성 · 규칙 검토

학급 전체 초안을 한 번에 생성하고 바이트 제한·기재 금지어를 자동으로 검토합니다.

검토 카드 치환

검증 이슈를 클릭하면 문제 문장이 하이라이트되고, 치환 버튼으로 즉시 수정됩니다.

개인정보 보호 설계

이름·전화번호·이메일 등 식별정보는 AI 전송 전 마스킹으로 최소화되고, 학생 자료는 KIBU 서버가 아닌 교사 PC에만 저장됩니다. API 키도 암호화되어 보관됩니다. 자세히

Why KIBU

기록 업무, 이렇게 달라집니다

기존 방식

  • 학생별 자료를 일일이 취합하고 복사·정리
  • 한 명씩 반복해서 문장 작성·타이핑
  • 나이스 바이트 수동 계산과 미세 조정
  • 기재 금지어를 눈으로 하나씩 확인
  • 자료가 뒤섞일까 늘 신경 쓰임

KIBU와 함께

  • 명렬표 + 자료를 한 번에 학생별 자동 배분
  • 학급 전체 초안을 클릭 한 번에 일괄 생성
  • 바이트·금지어를 자동으로 검증
  • 근거 자료를 하이라이트하는 검토 카드
  • 학생별 자료가 완전히 분리되어 안전

Why not just a chatbot

그냥 챗봇에 붙여넣는 것과 무엇이 다른가요?

학생 자료를 챗봇에 그대로 붙여넣는 방식은 빠르지만, 개인정보와 사실 검증에서 위험이 큽니다.
KIBU는 근거 → 검증 → 개인정보 최소화 → 교사 확정으로 이어지는 흐름을 도구 안에 담았습니다.

챗봇에 그대로 붙여넣기

  • 학생 이름·연락처가 그대로 외부 AI로 전송
  • 자료에 없는 내용을 그럴듯하게 지어낼 위험
  • 금지어·바이트·기재 규정은 직접 확인
  • 무엇이 어디로 전송됐는지 남는 기록 없음

KIBU의 검증된 흐름

  • 이름·전화·이메일은 전송 전 마스킹으로 최소화
  • 자료 근거가 없으면 쓰지 않도록 설계
  • 금지어·바이트·규칙을 자동으로 검토
  • 사용 모델·근거·검토 결과가 이력에 저장

자료에 근거해서만

제출된 자료에 있는 내용만 쓰도록 설계해, AI가 없는 사실을 지어내는 것을 막습니다.

개인정보는 최소화

이름·연락처 등 식별정보는 AI로 보내기 전에 자동으로 가려집니다.

여러 번 검토

근거 대조, 금지어, 규칙, 바이트를 여러 단계로 나눠 겹겹이 확인합니다.

최종 결정은 선생님

AI는 초안을 제안할 뿐, 검토 카드로 확인하고 확정하는 것은 언제나 교사입니다.

이 설계 방식은 국가 개인정보·AI 보안 가이드라인과 교원 AI·디지털 교육 역량체계가 강조하는 방향과 맞닿아 있습니다.
