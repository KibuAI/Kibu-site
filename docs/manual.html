KIBU 사용 설명서

KIBU AI의 설치부터 나이스 엑셀 내보내기까지 완벽 가이드입니다.

1. 시작하기

KIBU를 처음 실행하고 API를 연동하여 사용할 준비를 하는 과정입니다.

KIBU 설치 및 실행

다운로드 받은 설치 파일(KibuBeta-win-Setup.exe)을 더블클릭하면 설치가 진행되고 KIBU가 바로 실행됩니다. 바탕화면과 시작 메뉴에 바로가기가 만들어집니다.

자동 업데이트: 한 번 설치하면 이후 새 버전이 나올 때 KIBU가 시작 시 자동으로 확인해 내려받은 뒤, 재시작 여부를 물어봅니다. 더 이상 홈페이지에서 수동으로 다시 받을 필요가 없습니다.
참고: 실행에 필요한 .NET Desktop Runtime 9.0은 설치 과정에서 자동으로 확인·설치되므로 별도로 설치할 필요가 없습니다.

API 키 준비 및 설정

KIBU는 선생님의 로컬 PC에서 구동되며, 보안을 위해 개인의 OpenAI 또는 Gemini API 키를 사용해야 합니다.

API 키 발급 방법: KIBU는 둘 중 하나의 키만 있어도 사용할 수 있습니다.
  • OpenAI 키(기본 권장): platform.openai.com에 로그인 후 [API keys → Create new secret key]에서 발급하고, 결제 수단을 등록(Billing)합니다. sk-로 시작하는 키를 복사해 붙여넣으세요.
  • Gemini 키: Google AI Studio에 로그인 후 [Get API key → Create API key]에서 발급합니다.
더 자세한 절차는 인터넷에 'OpenAI API 키 발급' 또는 'Gemini API 키 발급'을 검색하시면 최신 화면과 함께 쉽게 확인하실 수 있습니다.
학교 예산 활용: 개인 카드 결제가 부담스럽다면 학교장터(S2B)의 에듀테크 업체를 통해 API 키(사용 권한)를 구매할 수 있습니다. 학교별 에듀테크 예산으로 결제할 수 있어, 교사 개인 비용 부담 없이 학교 단위로 도입·활용하기에 적합합니다.
무료 티어(Free Tier) 사용 주의사항: 발급받은 API가 무료 계정 상태일 경우, 제공 업체의 분당 요청 횟수(Rate Limit) 제한에 걸려 KIBU의 다중 학생/다중 자료 동시 분석이 정상적으로 완료되지 않고 오류가 발생할 수 있습니다. 원활한 대량 작업과 빠른 속도를 위해서는 결제 수단을 연동(Pay-as-you-go)하여 제한을 해제하신 후 사용하시는 것을 권장합니다.

API 연동 화면에서 키 입력하기

메인 화면 우측 상단의 [설정] 버튼을 눌러 [API 연동] 화면을 엽니다. 발급받은 키를 붙여넣고 저장하면 준비가 끝납니다.

화면 구성과 입력 방법
  • OpenAI API 키 / Gemini API 키: 각 칸에 발급받은 키를 붙여넣습니다. 둘 중 하나만 넣어도 동작하며, 오른쪽 눈 모양() 아이콘으로 입력한 키를 확인할 수 있습니다.
  • 안전한 저장(DPAPI): 입력한 키는 Windows DPAPI로 현재 PC 사용자 계정에 암호화되어 보관됩니다. 외부로 전송되거나 평문으로 노출되지 않습니다.
  • [모델 목록 새로고침]: 상단 버튼을 누르면 입력한 키로 실제 사용 가능한 최신 모델 목록을 불러옵니다. (자세한 동작은 아래 참고)
  • [고급 모드: 단계별 모델 직접 지정] 토글: 켜면 기본 모델 대신 아래 [단계별 모델 직접 지정]에서 고른 모델이 최우선으로 적용됩니다. 제미나이만 쓰거나 단계별로 모델을 바꾸고 싶을 때 사용합니다.
  • 저장: 마지막으로 하단의 [저장] 버튼을 눌러야 설정이 적용됩니다.

아래는 모델 구성·새로고침·비용 등 API에 대한 자세한 안내입니다.

기본 AI 모델 구성 (OpenAI 키 기준): KIBU는 설치 직후 별도 설정 없이도 각 단계에 최적화된 OpenAI 모델이 자동으로 지정됩니다. 자동 생성 프롬프트와 처리 프로세스도 이 기본 구성에 맞춰 최적화되어 있습니다.
  • 자료 판독(스캔본·이미지)·초안 생성: GPT-5.4 — 손글씨 지도/보고서 판독과 근거 기반 서술에 필요한 높은 성능
  • 문체 정제·규정 검증: GPT-5.4-mini — 텍스트 정리에 특화된 경량 모델로 품질은 유지하면서 비용을 절감
메인 화면 우측 상단의 [설정] 버튼 → [AI 모델]에서 단계별 모델을 언제든 자유롭게 변경할 수 있습니다. 일반적으로 더 고급 모델을 사용할수록 학술적이고 정교한 표현이 등장할 확률이 높아집니다.
제미나이(Gemini) 사용 시: 기본 구성은 OpenAI 기준이므로, 제미나이 키만 사용하시려면 [설정 → 고급 모드]를 켠 뒤 모든 단계를 제미나이 모델로 직접 지정해야 합니다. (제미나이도 스캔본 이미지 판독을 지원합니다.)
손글씨·스캔본 판독은 상위 모델 권장: 제미나이로 손글씨 원본이나 스캔 PDF를 판독할 때는 Gemini 3.1 Pro (Preview)처럼 고급(Pro) 모델을 사용해야 인식 정확도가 확보됩니다. Flash 등 경량 모델은 빠르고 저렴하지만 흐린 손글씨나 복잡한 표를 잘못 읽을 수 있으므로, 자료 판독 단계만큼은 상위 모델로 지정하시는 것을 권장합니다.
모델 목록 새로고침: [설정 → AI 모델] 화면의 [모델 새로고침] 버튼을 누르면, 입력한 API 키로 제공사(OpenAI·Google) 서버에 직접 조회하여 그 키로 실제 사용할 수 있는 최신 모델 목록을 불러옵니다. 새 모델(예: 새로 출시된 Gemini Pro, GPT 신규 버전)이 목록에 보이지 않을 때 이 버튼을 눌러 주세요.
  • 새로고침 시 입력한 OpenAI 키·제미나이 키가 함께 저장된 뒤 각 키에 맞는 모델을 조회합니다. (키를 넣은 제공사만 조회)
  • 인터넷이 잠시 불안정하거나 키가 잘못되어 조회에 실패해도, 기본 내장 모델 목록은 그대로 유지되어 작업에는 지장이 없습니다.
  • 목록에 원하는 모델이 안 보이면, 대개 해당 키의 결제/사용 권한이 없거나 아직 그 계정에 개방되지 않은 모델입니다.
예상 비용 안내: 기본 모델 구성 기준, 학생 1명의 초안을 한 번 생성할 때 첨부한 자료의 양과 스캔본 여부에 따라 약 200~500원 정도의 API 사용료가 발생합니다. 자료가 적으면 더 저렴하고, 스캔본이 많거나 분량이 큰 경우 비용이 다소 올라갑니다. 사용료는 선택한 모델에 따라 달라지며, 일반적으로 제미나이(Gemini) 계열이 OpenAI보다 저렴한 편이라 비용을 더 아끼고 싶다면 고급 모드에서 제미나이로 전환하는 것도 방법입니다. (다만 모델별로 문체·판독 품질 차이가 있을 수 있습니다.)

외부 AI 이미지 분석 허용

손글씨나 스캔 PDF 등 이미지 기반 문서를 판독하려면, 설정 화면 아래쪽의 [외부 AI 이미지 분석 허용]을 ON으로 켜야 합니다.

  • ON: 정밀 판독을 위해 원본 이미지를 선택한 AI로 전송하여 분석합니다. (손글씨·스캔본을 다룬다면 반드시 켜 주세요.)
  • OFF(기본): 원본 이미지는 전송하지 않고 로컬 OCR로 뽑은 텍스트만 사용합니다.

2. 프로젝트 만들기

과목과 학급 단위로 작업 공간(프로젝트)을 생성합니다.

3. 자료 가져오기 (자동 배분)

학생 목록과 평가 자료를 연동하는 핵심 단계입니다.

명렬표 가져오기

나이스에서 다운로드한 교과출석부 엑셀 파일을 그대로 업로드하면 학생 목록이 자동으로 생성됩니다.

자료 유형별 가져오기 및 자동 배분 방식

[자료 가져오기]에서 설문 응답 엑셀 파일이나 여러 학생의 활동지가 스캔된 대용량 PDF를 선택하세요. KIBU는 평소 사용하시던 다양한 형태의 파일을 그대로 인식하여 자동으로 학생별 보관함에 분류합니다.

엑셀 파일
구글/네이버 폼 응답

작동 방식: 표의 최상단에서 '학번', '이름' 열(Column)을 자동으로 감지하여 각 행(Row)의 긴 텍스트 응답을 해당 학생에게 즉시 분배합니다.

워드 문서
학습 일지, 독후감

작동 방식: 문서 본문 속에 적혀있는 '학번'이나 '이름' 관련 패턴을 AI가 탐색하여 파일 전체의 내용을 알맞은 학생에게 통째로 매핑합니다.

스캔 PDF/이미지
종이 활동지, 시험지

작동 방식: OCR(문자인식)을 거쳐 페이지 상단의 손글씨 학번/이름을 분석합니다. 자동 인식이 불가능할 땐 수동 배정 기능으로 묶어서 배분할 수 있습니다.

KIBU가 파일 내의 학번과 이름을 인식하여 자료를 학생별로 자동 배분해 줍니다.

대용량 엑셀 파일 업로드 안내: 100명 이상의 학생 응답이 포함된 대규모 설문 엑셀 파일 등을 업로드할 경우, 학생별로 데이터베이스(DB)를 생성하고 기록을 분배하는 과정에서 수 분 정도 프로그램이 멈춘 것처럼(로딩 중) 보일 수 있습니다. 이는 오류가 아닌 정상적인 데이터 처리 과정이므로, 처리가 완료될 때까지 잠시만 기다려주시면 됩니다.
스캔본 배정 안내: AI가 학번과 이름을 완벽하게 인식하기 어려운 스캔본(PDF)의 경우, 화면 우측의 [배정 정보 수정] 기능을 이용하여 각 페이지를 특정 학생에게 직관적으로 배정할 수 있습니다.

학생별 파일을 직접 끌어다 넣기 (개별 자료함 드래그 앤 드롭)

이미 학생 목록이 만들어진 상태(명렬표 가져오기 등)이고, 학생별로 파일을 따로 가지고 계신다면, [자료 가져오기]로 일일이 배정할 필요 없이 원하는 학생의 개별 자료함에 파일을 바로 끌어다 놓을 수 있습니다. 특정 학생 몇 명만 자료를 보충하거나, 학번/이름이 파일에 안 적혀 있어 자동 배분이 어려운 경우에 특히 편리합니다.

사용 방법 (3단계)
  1. 학생 선택: [학생 패널]에서 자료를 넣을 학생을 클릭해 선택합니다. (학생을 먼저 선택하지 않으면 "자료를 추가할 학생을 먼저 선택하세요" 안내가 나옵니다.)
  2. 영역 선택: 상단에서 자료를 넣을 영역(예: 교과세특, 진로활동 등)을 선택합니다. 그 학생의 해당 영역 개별 자료함으로 들어갑니다.
  3. 드래그 앤 드롭: 워드·엑셀·이미지·PDF 등 파일을 탐색기에서 끌어와 개별 자료함 영역에 놓으면, KIBU가 내용을 읽어 그 학생의 자료로 추가합니다.
  • 스캔본·이미지도 바로 사용: 텍스트가 없는 스캔 PDF나 사진(JPG·PNG)도 개별 자료함에 그대로 끌어다 놓을 수 있습니다. 원본이 학생 자료로 등록된 뒤 초안 생성 시점에 AI가 이미지를 판독(OCR·비전 분석)하여 반영하므로, 드롭만 해두면 별도의 마법사 없이 바로 초안 생성에 쓰입니다. (손글씨 인식 정확도를 높이려면 자료 판독 단계를 상위 모델로 지정하세요.)
  • 여러 학생 공통 자료: 학급 전체에 해당하는 안내문·수업 자료 등은 학생 선택 없이 공통 자료함에 드롭하면 모든 학생 초안에 참고 자료로 활용됩니다.
  • 파일명 기반 영역 확인(안전장치): 파일명에 자율·진로·동아리·세특·행발 같은 영역 단어가 들어 있는데 현재 선택한 영역과 다를 때만, 실수 방지를 위해 확인 창을 한 번 띄웁니다. (이런 키워드가 없는 일반 파일명은 확인 창 없이 바로 등록됩니다.)
  • 많은 학생에게 한꺼번에 배분할 때: 한 파일 안에 여러 학생 자료가 섞여 있어 학번·이름으로 자동 분류가 필요하다면, 드래그 대신 [자료 가져오기] 마법사를 쓰는 것이 편리합니다. (드래그는 "지금 선택한 한 학생"에게 넣는 방식입니다.)

4. 맞춤형 프롬프트

선생님의 교육 철학과 과목의 특성을 생기부에 담아내어 AI에게 학습시킵니다.

  • 평가 관점 선택: 강조하고 싶은 핵심 역량(태도, 문제해결력, 의사소통 등)을 체크합니다.
  • 교사 요청사항 (주안점): 이번 수업이나 수행평가에서 특별히 부각하고 싶은 학생의 탐구 활동, 성장 과정, 또는 교과 특유의 관점(예: "데이터 분석 과정에서의 논리적 사고력을 중심으로 작성할 것")을 구체적으로 입력하세요.

5. 초안 생성

개별 생성 vs 학급 전체 일괄 생성

특정 학생만 확인하고 싶을 때는 상단 작업 흐름의 [✨ AI 개별 초안 생성] 버튼을, 전체 준비가 끝났을 때는 우측 상단 [전체 작업] → [학급 전체 일괄 생성]을 클릭하세요. 자동 검증 체크는 [AI 개별 초안 생성] 버튼 오른쪽에 있으며, 개별·전체 생성에 함께 적용됩니다. 초안이 생성되면 화면은 자동으로 [AI 초안 에디터] 패널로 전환됩니다.

자동 검증 옵션(켜기/끄기): 자동 검증을 켜면 초안 생성 시 학생 자료와 초안의 정합성 + 기재금지어 등 규칙 검사를 함께 수행합니다. 맞춤형 프롬프트로 나오는 초안 품질만 빠르게 확인하고 싶을 때는 자동 검증을 꺼도 됩니다. (검증은 나중에 검토 카드나 기재규칙검사로 다시 돌릴 수 있습니다.)

6. 검토와 수정 (검토 카드)

AI가 작성한 내용을 최종적으로 선생님이 확정하는 단계입니다.

  • 불확실 판독 확인: 손글씨가 너무 흐려 확신할 수 없는 단어는 노란색으로 하이라이트됩니다. 반드시 선생님의 눈으로 다시 확인해 주세요.
  • 바이트 확인: 수정 과정에서도 실시간으로 바이트를 체크합니다.

목표 바이트(글자 수) 조정과 동작 방식

초안 에디터 상단의 목표 바이트 값은 그 학생·영역의 최종 분량 한도입니다. 영역을 선택하면 표준값(교과세특·창의적체험활동 1,500B, 행동특성및종합의견 900B)이 자동으로 채워지며, 필요할 때 직접 숫자를 바꿀 수 있습니다.

값을 수정하면 이렇게 동작합니다.
  • 수동 조정 모드로 전환: 표준값과 다른 숫자를 입력하면 그 값이 기준 한도로 고정됩니다. (표준값으로 되돌리려면 [영역 기본값으로 초기화]를 누르세요.)
  • 실시간 게이지 갱신: 현재 글자 수가 한도를 넘으면 경고 표시가, 권장 하한(한도의 약 80%) 아래로 너무 짧으면 "분량 부족" 안내가 즉시 나타납니다.
  • 초안 생성 목표에 반영: 새로 초안을 생성할 때 AI는 정제 여유분을 감안해 한도의 약 1.2배(행발은 1.3배)를 상한 목표로 넉넉히 쓴 뒤, 정제 단계에서 최종 한도 이내로 줄입니다. 그래서 값을 올리면 더 길고 풍부하게, 내리면 더 압축된 초안이 나옵니다.
  • 기재규칙검사에 반영: 검사 시 최종 글자 수가 한도를 넘으면 [BYTE_LIMIT] 검토 카드로 경고합니다.
  • 기존 본문은 자동으로 잘리지 않습니다. 목표 바이트를 낮춰도 이미 작성된 글은 그대로 유지되며, 줄이는 것은 선생님의 판단에 맡깁니다. (미세한 1~2바이트 차이는 조사·수식어 하나만 다듬으면 맞춰집니다.)

여러 명 한 번에 검사할 때(전체 기재규칙검사)의 동작

[전체 기재규칙검사]는 학생을 한 명씩 독립적으로 검증합니다. 각 학생마다 설정된 검증 모델로 별도의 AI 호출이 이루어지므로, 한 학생의 결과가 다른 학생에게 섞이지 않습니다.

"끊기듯 진행되는" 느낌은 정상입니다.
  • 동시 3명 + 페이싱 간격: API 사용량 제한(Rate Limit)에 걸리지 않도록 최대 3명씩 병렬로 처리하고, 각 처리 사이에 짧은 간격을 둡니다. 그래서 진행 표시줄이 3명 단위로 멈췄다 이어지듯 보이는데, 중단이 아니라 안전을 위한 정상 동작입니다.
  • 자동 재시도·대체: 일시적인 네트워크·서버 혼잡(타임아웃, 429, 5xx)이 발생하면 같은 모델로 지수적 간격을 두고 자동 재시도하고, 그래도 안 되면 보조 모델로 넘겨 최대한 완료합니다.
  • 실패한 학생 확인: 재시도로도 검증 호출이 끝내 실패한 학생은 검토 카드에 "검증 AI 호출 실패"로 남습니다. 이 학생만 잠시 후 개별 [기재규칙 재검사]로 다시 돌리면 됩니다.
  • 무료 티어 주의: 무료 계정 키는 분당 요청 제한이 낮아 대량 검사 중 실패가 잦을 수 있습니다. 결제(Pay-as-you-go)를 연동하면 훨씬 안정적으로 완료됩니다.

완성된 초안(엑셀)을 불러와 규칙 검사만 하기

이미 작성해 둔 생기부 초안이 엑셀(학번·이름·초안 내용 등)로 있다면, 초안을 새로 생성하지 않고 기재규칙검사만 돌려 금지어·표현·바이트를 빠르게 점검할 수 있습니다.

엑셀 초안 → 규칙 검사 (3단계)
  1. 자료 가져오기: 학번·이름과 초안 열이 담긴 엑셀을 [자료 가져오기]로 불러오면, 각 학생의 개별 자료함에 열 단위로 저장됩니다.
  2. 검사 실행: 초안 에디터가 비어 있는 상태에서 [기재규칙검사](현재 학생) 또는 [전체 작업 → 전체 기재규칙검사](학급 전체)를 누릅니다. "초안 에디터가 비어 있습니다. 개별 자료함의 데이터를 불러오시겠습니까?"에 [예]를 선택합니다.
  3. 열 선택: 자료함에 있는 열(예: 교과세특 초안) 목록이 뜨고, 각 열마다 해당 학생 수·항목 수·미리보기가 표시됩니다. 초안으로 쓸 열을 고르면 각 학생 초안 영역이 채워지고 곧바로 규칙 검사가 진행됩니다.
참고
  • 기존 초안 덮어쓰기 옵션을 켜면 이미 초안이 있는 학생도 자료함 내용으로 교체됩니다(끄면 빈 학생만 채움). 덮어쓸 학생이 있으면 한 번 더 확인합니다.
  • "불러올 자료가 없습니다" 안내가 뜨면, 먼저 [자료 가져오기]로 학생별 초안/자료를 넣어 주세요.
  • 불러온 내용은 현재 선택한 영역의 초안으로 들어갑니다. 실행 전 상단의 영역 선택이 맞는지 확인하세요.

7. 내보내기

모든 검토가 끝난 학생들의 기록은 엑셀 내보내기를 통해 일괄 저장할 수 있습니다. 저장된 파일은 나이스에 바로 복사-붙여넣기 하기 좋은 형태로 가공되어 있습니다.

8. 고급 설정

AI 모델 유연한 전환 (OpenAI ↔ Gemini)

작업의 성격이나 선생님의 선호도에 따라 주력으로 사용할 AI 모델을 언제든지 변경할 수 있습니다. 설정 화면에서 각 단계별(초안 생성, 검토, 분석 등)로 가장 적합한 모델을 선택하여 유연하게 활용해 보세요.

규칙 관리 (금지어 및 바이트 검증)

학교마다 학교생활기록부에 기재해서는 안 되는 금칙어(예: 외부 대회명, 사설 기관명, 부모 직업 등)나, 과목별로 정해진 바이트 제한이 다를 수 있습니다. 메인 화면 우측 상단의 [규칙 관리자] 버튼을 눌러 우리 학교만의 금지 단어 목록과 바이트 상한선을 미리 등록해 두세요. 초안을 생성하거나 검토할 때 KIBU가 해당 규칙을 위반한 곳이 없는지 실시간으로 꼼꼼하게 검증하여 경고해 줍니다.

① 감지할 표현을 입력하는 두 가지 방법

규칙 하나를 만들 때, 무엇을 잡아낼지 지정하는 방법은 두 가지입니다.

  • 일반 단어·문구 입력 (기본): [교사용 표시 문구] 칸에 잡고 싶은 단어를 그대로 적으면 됩니다. (예: 뛰어난, 최고의, 완벽한) 정규식 같은 전문 지식이 필요 없어 대부분의 규칙은 이 방법으로 충분합니다.
    • 비슷한 표현도 함께 감지 (어미 변화 포함): 이 옵션을 켜면 뛰어난 하나만 적어도 뛰어나다, 뛰어나며처럼 어미가 바뀐 형태까지 자동으로 함께 잡아 줍니다.
  • 정규식(Regex) 직접 입력 (고급): [고급 설정 표시]를 켜면 입력 모드를 정규식으로 바꾸고 감지 패턴을 직접 작성할 수 있습니다. 복잡한 조건(숫자+단위, 특정 형식 등)을 정밀하게 잡을 때 사용합니다.
규칙 작성이 어렵다면: 정규식을 직접 만들기 어렵다면 [AI로 규칙 만들기] 버튼을 누르고 원하는 규칙을 평소 말하듯 문장으로 설명해 주세요. (예: "교외 수상 실적이 들어가면 잡아 줘") KIBU가 알맞은 감지 규칙을 대신 만들어 목록에 추가해 줍니다.

② 걸린 표현을 어떻게 처리할지 (두 가지 방식)

규칙마다 위반 표현을 발견했을 때의 처리 방식을 아래 두 가지 중에서 선택합니다.

  • 📋 검토 카드에 표시 (교사 확인): 자동으로 바꾸지 않고, 검토 패널에 이슈 카드로 띄워 선생님이 직접 확인하고 수정합니다. 판단이 필요한 대부분의 규칙(과장 표현, 주관적 서술 등)은 이 방식이 가장 안전합니다.
  • 🔄 형식·표현 자동 정리 (대체어 지정): 걸린 표현을 미리 정해 둔 단어로 초안 생성 시 자동으로 바꿔 줍니다. (예: 체육대회 → 체육활동) 문맥을 따질 필요 없이 단순 치환이 명확할 때만 사용하세요.
    • 대체어를 비워 두면 자동으로 바꾸지 않고 검토 카드에만 표시됩니다.
어떤 방식을 고를까? 자연스럽게 바꿀 대체어가 명확하게 하나로 정해질 때만 '자동 정리'를 쓰고, 조금이라도 판단·문맥이 필요하면 '검토 카드에 표시'를 선택하세요. 잘못된 자동 치환보다 교사 확인이 안전합니다.

③ 규칙을 더 정교하게 다듬기

  • 심각도: 금지(반드시 걸러야 하는 표현) · 주의 · 참고로 나눠, 위반의 경중을 구분해 표시합니다.
  • 적용 범위: 규칙을 전체에 적용할지, 교과세특·진로활동·자율활동·행동특성및종합의견 중 특정 영역에만 적용할지 지정할 수 있습니다.
  • 규칙 테스트: 예시 문장을 입력하면 이 규칙이 실제로 무엇을 잡아내는지 저장 전에 미리 확인할 수 있습니다.
  • 교사용 설명: "왜 조심해야 하는지"를 적어 두면 검토 카드에 함께 표시되어, 나중에 이유를 다시 떠올리기 쉽습니다.

초안을 엑셀 등으로 불러온 뒤 '기재규칙검사'만 단독으로 돌려, 금지어·표현만 빠르게 점검할 수도 있습니다.

안전한 생기부 작성: 사람이 놓치기 쉬운 규정 위반 단어나 초과된 바이트를 시스템이 한 번 더 걸러주므로, 나이스 최종 입력 시 발생할 수 있는 반려나 수정 사유를 사전에 완벽히 차단할 수 있습니다.

9. 문제 해결 (FAQ)

사용 중 마주칠 수 있는 상황과 해결책을 안내합니다. 각 항목을 클릭하면 답변이 펼쳐집니다.

구독제가 아닙니다. 전기·수도처럼 쓴 만큼만 API 사용료가 나가고, 쓰지 않으면 0원입니다.
  • 대략 학생 1명 초안 1회 약 200~500원, 30명 한 반이면 약 6천~1만 5천 원 수준입니다. (자료 양·스캔본 여부에 따라 달라짐)
  • 선불 충전 + 사용 한도(limit) 설정으로 '요금 폭탄'을 막을 수 있습니다.
  • 결제(카드) 정보는 OpenAI / 구글에만 등록되며, KIBU는 결제 정보를 저장하지 않습니다.
  • 입문 단계에서는 일반적으로 Gemini 계열이 더 저렴한 편입니다.
  • 개인 결제가 부담된다면 학교장터(S2B)의 에듀테크 업체를 통해 학교 에듀테크 예산으로 API 키를 구매·도입할 수도 있습니다.

섞이지 않습니다. 여러 명을 동시에 처리하더라도 학생마다 완전히 분리된 처리 공간이 새로 만들어지고, 자료는 각 학생의 학번을 기준으로만 조회·연결됩니다. 서로의 자료나 초안이 공유되는 구조가 아니며, 저장 역시 순차적으로 처리되어 데이터가 뒤섞이지 않도록 설계되어 있습니다.

저장은 100% 로컬입니다. KIBU는 자체 서버를 운영하지 않으며, 학생 자료와 생성된 초안은 오직 선생님의 PC(로컬 SQLite DB)에만 저장됩니다.

다만 초안 생성 자체는 AI 처리를 위해 선생님이 등록한 개인 OpenAI/Gemini 키를 통해 해당 회사 서버(국외)로 전송됩니다(이는 모든 생성형 AI 도구의 공통된 구조입니다). 이 과정에서 이름·전화번호·이메일 등 직접 식별정보는 전송 전 마스킹 규칙으로 최소화되지만, 활동 내용 본문 안에 적힌 개인정보까지 완벽히 제거되지는 않으므로 업로드 자료에는 꼭 필요한 정보만 남겨 주세요.

스캔본 이미지(손글씨 원본) 자체는 기본적으로 외부에 전송되지 않고 로컬 OCR 텍스트만 사용되며, [설정 → 외부 이미지 분석]을 직접 켠 경우에만 정밀 판독을 위해 이미지가 AI사로 전송됩니다.

자세한 내용은 개인정보 처리 및 AI 이용 안내 페이지를 확인해 주세요.

네, 사용할 수 있습니다. 다만 기본 구성이 OpenAI 기준이므로, [설정 → 고급 모드]를 켜고 모든 단계(자료 판독, 초안 생성, 정제, 검증)를 제미나이 모델로 지정하면 OpenAI 키 없이도 이용할 수 있습니다. 제미나이도 스캔본 이미지 판독을 지원합니다.

여러 자료를 동시에 분석하고 맥락을 연결하는 One-Pass 과정은 일반 요약보다 많은 연산이 필요합니다. 한 명씩 기다리지 마시고 우측 상단 [전체 작업] → [학급 전체 일괄 생성]을 클릭한 뒤, 백그라운드에서 동작하게 두고 다른 업무를 보시는 것을 권장합니다.

구글이나 OpenAI 서버의 일시적인 혼잡으로 인한 오류입니다. 모델 설정에서 [새로고침]을 누르시거나, 1~2분 정도 대기 후 다시 시도하시면 정상적으로 작동합니다.

파일의 용량이 비정상적으로 크거나 화질이 너무 낮을 수 있습니다. 불필요한 페이지가 많은 PDF라면 필요한 페이지(예: 1~3페이지)만 분리해서 업로드해 주세요. 또한 설정에서 [외부 이미지 분석] 옵션이 ON으로 되어있는지 확인 바랍니다.

AI는 추상적인 지시보다 구체적인 평가 맥락을 줄 때 가장 잘 작동합니다. "교사 요청사항" 칸에 단순히 "잘 써줘"라고 하기보다는, "실험 과정에서 실패를 극복하고 원인을 분석해가는 태도에 주안점을 두고 작성해줘"와 같이 어떤 점을 중점적으로 평가했는지 상세히 적어주시면 훨씬 만족스러운 결과를 얻을 수 있습니다.

AI가 생성하는 텍스트의 특성상 설정한 바이트에 100% 완벽하게 떨어지지 않고 약간의 오차가 발생할 수 있습니다. 이 경우 검토 카드 화면에서 불필요한 조사나 단어 하나를 직접 수정하시면 실시간으로 바이트가 맞춰집니다.

윈도우 환경에 .NET Desktop Runtime 9.0이 정상적으로 설치되지 않았을 가능성이 높습니다. 마이크로소프트 공식 홈페이지에서 해당 런타임을 재설치한 후 프로그램을 관리자 권한으로 실행해 보시기 바랍니다.

입력하신 API 키의 앞뒤에 공백이 포함되어 있지 않은지 확인해 주세요. 또한, 해당 API 계정의 결제 티어(Billing)가 만료되었거나 한도를 초과했을 수 있으니 구글/OpenAI 개발자 콘솔에서 상태를 점검하시기 바랍니다.

10. 고급 사용자 팁 (Advanced)

압축을 풀었을 때의 폴더 구조와 프로그램을 좀 더 세밀하게 다루고자 하는 분들을 위한 KIBU 활용 팁입니다.

1. 데이터 저장 위치와 백업

설치형 KIBU는 학생 목록·설정·생성된 초안 등 모든 사용자 데이터를 %LocalAppData%\Kibu 폴더에 저장합니다. (탐색기 주소창에 %LocalAppData%\Kibu를 붙여넣으면 바로 열립니다.)
이 폴더는 앱 프로그램 파일과 분리되어 있어, 자동 업데이트로 앱이 갱신되어도 데이터는 그대로 유지됩니다.
환경을 통째로 백업하거나 다른 PC로 옮기고 싶다면 이 Kibu 폴더를 복사해 두었다가 같은 위치에 붙여넣으면 됩니다. 다만 학기·연도 단위 보관에는 아래의 프로젝트 폴더 백업 방식을 권장합니다.

2. KIBU 프로젝트 폴더 전체 백업하기

KIBU에서 '새 프로젝트'를 만들면 선생님이 지정한 위치에 프로젝트 폴더가 생성되며, 그 내부에 project.kibu(데이터베이스) 파일과 업로드했던 원본 파일, 분석 캐시 데이터들이 모두 저장됩니다.
작업 내역을 학기나 연도 단위로 영구 보관하고 싶다면 이 프로젝트 폴더 전체를 통째로 복사하거나 압축해서 백업해 두시는 것을 권장합니다. 나중에 KIBU를 실행하고 [프로젝트 열기]에서 해당 폴더의 project.kibu를 선택하기만 하면 과거의 모든 원본 자료와 작업 기록이 완벽히 복원됩니다.

3. 기본 프롬프트·규칙 커스터마이즈

과목·학교 사정에 맞게 프롬프트나 기재 규칙을 바꾸고 싶다면 앱 안의 프롬프트 관리와 규칙 관리 화면에서 직접 편집·저장하시면 됩니다. 여기서 만든 사용자 프롬프트와 규칙은 %LocalAppData%\Kibu에 저장되어 자동 업데이트가 되어도 그대로 보존됩니다.
기본 제공 프롬프트 템플릿은 각 버전의 Defaults\Kibu_DefaultPrompts.json에 포함되어 배포되며, 새 버전이 나오면 기본값만 갱신되고 선생님이 만든 항목은 유지됩니다.

4. 외부 플러그인 (Pdfium)

폴더에 함께 있는 pdfium.dll 파일은 스캔 문서(PDF) 분석을 위한 핵심 엔진입니다. 이 파일이 삭제되거나 격리되면 스캔본 분석 기능이 작동하지 않으니 임의로 삭제하지 않도록 주의하세요.
